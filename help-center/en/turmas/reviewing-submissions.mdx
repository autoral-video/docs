---
title: "Reviewing submissions"
description: "Use the review queue to grade student submissions, give feedback, assign scores and request revisions."
---

The **review queue** brings together all student submissions for you (or the mentors) to grade in one place.

Access it in **Admin → Classes → [Class] → Reviews**.

```
/admin/classes/{class}/review-queue
```

## Grading a submission

When you open a submission, you see the submitted content next to the grading form:

| Field | Description |
|---|---|
| **Feedback** | Comment for the student (optional). |
| **Status** | Result of the review (see below). |
| **Score** | Score from 0 to 100 (optional). |

## Review status

| Status | Meaning |
|---|---|
| **Pending** | Awaiting review. |
| **In review** | Being reviewed. |
| **Approved** | Submission accepted. |
| **Needs revision** | Requires adjustments; the student can resubmit (if resubmission is allowed). |
| **Rejected** | Not accepted. |

## Filters and organization

- Filter by **assignment** and by **status**
- Counters show how many submissions are pending, approved, needing revision, and so on.

The student is notified when the review is published and can see the feedback and score. If the status is **needs revision**, they can resubmit.
