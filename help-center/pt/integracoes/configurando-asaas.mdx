---
title: "Configurando o Asaas"
description: "Configure o Asaas via chave de API para aceitar cartão, PIX, boleto e assinaturas recorrentes."
---

O Asaas é um gateway brasileiro que oferece uma ampla variedade de métodos de pagamento, incluindo cartão de crédito, PIX, boleto bancário e assinaturas recorrentes. É uma excelente opção para diversificar as formas de pagamento aceitas pelos seus alunos.

A integração com o Asaas é feita por **chave de API**, sendo necessário copiar duas credenciais do painel do Asaas: a chave de API e o token do webhook.

## Métodos de pagamento disponíveis

- **Cartão de crédito** — com parcelamento
- **PIX** — pagamento instantâneo
- **Boleto bancário** — com vencimento configurável
- **Assinaturas recorrentes** — cartão, PIX e boleto

## O que você vai precisar

Antes de começar, certifique-se de ter:

- Uma conta no Asaas — crie em [asaas.com](https://www.asaas.com)
- Acesso ao painel do Asaas para gerar credenciais

## Passo 1: Gere a chave de API no Asaas

1. Faça login no painel do Asaas
2. Acesse **Configurações → Integrações → API**
3. Gere uma nova **chave de API**
4. Copie a chave gerada — você vai usá-la no próximo passo

<Note>Para testes, gere a chave no ambiente **Sandbox**. Para receber pagamentos reais, use o ambiente de **Produção**.</Note>

## Passo 2: Configure o Asaas na plataforma

1. Acesse **Admin → Configurações → Asaas**
2. Em **Configurações gerais**, selecione o **Ambiente** correspondente à chave gerada:
   - **Sandbox (testes)** — para validar a integração sem cobrança real
   - **Produção** — para receber pagamentos reais
3. Defina os **Dias de vencimento do boleto (padrão)** entre 1 e 30 dias
4. Em **Credenciais da API**, cole sua chave no campo **Chave de API**
5. Clique em **Salvar**

<Note>O campo de chave de API mostra apenas pontos quando uma chave já está configurada. Para substituí-la, basta colar a nova chave e salvar. Para mantê-la, deixe o campo em branco.</Note>

## Passo 3: Configure o webhook no Asaas

O webhook é como o Asaas avisa a plataforma sobre o status dos pagamentos (aprovado, recusado, reembolsado, etc.).

1. Na tela de configuração do Asaas na plataforma, **copie a URL do webhook** exibida (algo como `https://seu-dominio.com/webhook/asaas`)
2. Acesse o painel do Asaas em **Configurações → Integrações → Webhook**
3. Cadastre a URL copiada como endpoint do webhook
4. Defina um **token de autenticação** no Asaas e copie-o
5. Volte à plataforma e cole o token no campo **Token do Webhook**
6. Clique em **Salvar**

<Warning>O token do webhook protege a comunicação entre o Asaas e a plataforma. Sem ele, eventos enviados pelo Asaas serão recusados.</Warning>

## Passo 4: Ative o Asaas

Após cadastrar a chave de API e o token do webhook, ative o toggle **Ativar Asaas** para habilitá-lo como forma de pagamento nos seus checkouts.

<Note>O Asaas só estará disponível no checkout após estar configurado e ativado.</Note>

## Alterando credenciais

Para substituir a chave de API ou o token do webhook:

1. Acesse **Admin → Configurações → Asaas**
2. Cole a nova credencial no campo correspondente
3. Clique em **Salvar**

Os campos exibem pontos quando uma credencial já está salva. Deixe o campo em branco para manter o valor existente, ou cole um novo valor para substituí-lo.

## Resumo das ações

| Ação | Onde |
|---|---|
| Gerar chave de API | Painel do Asaas → Configurações → Integrações → API |
| Cadastrar webhook | Painel do Asaas → Configurações → Integrações → Webhook |
| Configurar credenciais | Admin → Configurações → Asaas → **Credenciais da API** |
| Definir ambiente e boleto | Admin → Configurações → Asaas → **Configurações gerais** |
| Ativar / desativar no checkout | Admin → Configurações → Asaas → toggle **Ativar Asaas** |
