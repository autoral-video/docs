---
title: "Listar webhooks"
api: "GET /v1/webhooks"
description: "Lista paginada (length-aware) de webhooks outgoing."
---

**Escopo:** `webhooks:read`

Webhooks **outgoing** são endpoints externos que a plataforma chama quando eventos ocorrem (pagamento aprovado, aluno matriculado, etc.).

<Note>
Para o guia completo — configuração no painel, lista de eventos com payloads e validação de assinatura — veja a Central de Ajuda: [Configurando webhooks](/help-center/pt/webhooks/configurando-webhooks), [Eventos disponíveis](/help-center/pt/webhooks/eventos-disponiveis) e [Segurança e entregas](/help-center/pt/webhooks/seguranca-e-entregas).
</Note>

## Query params

<ParamField query="per_page" type="integer" default="25">1-100.</ParamField>
<ParamField query="active" type="boolean">Filtra ativos/inativos.</ParamField>

## Response

<ResponseField name="data[].id" type="integer">ID.</ResponseField>
<ResponseField name="data[].name" type="string">Nome.</ResponseField>
<ResponseField name="data[].url" type="string">Endpoint HTTPS.</ResponseField>
<ResponseField name="data[].events" type="array<string>">Eventos assinados.</ResponseField>
<ResponseField name="data[].is_active" type="boolean">Ativo.</ResponseField>

<RequestExample>
```bash cURL
curl https://seu-dominio/api/v1/webhooks \
  -H "Authorization: Bearer aut_..."
```
</RequestExample>

<ResponseExample>
```json 200
{
  "data": [
    {
      "id": 1,

      "url": "https://n8n.example.com/webhook/autoral",
```

</ResponseExample>

<Note>
  O **secret** do webhook nunca é retornado pela API — fica encriptado no banco. Para regenerar, delete e crie outro.
</Note>
