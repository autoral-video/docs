---
title: "Criar lead"
api: "POST /v1/leads"
description: "Captura um lead vindo de formulário externo."
---

**Escopo:** `leads:write`
**Idempotency:** suportada

Use para integrar formulários do seu site/landing page com a plataforma. O lead é criado com `status=pending` e fica disponível para sequências de e-mail e automações de carrinho abandonado.

## Body params

<ParamField body="checkout_id" type="integer" required>
  ID do checkout associado (deve pertencer à tenant).
</ParamField>

<ParamField body="name" type="string" required>1-120 chars.</ParamField>

<ParamField body="email" type="string" required>Email válido (max 180).</ParamField>

<ParamField body="phone" type="string">Max 32 chars.</ParamField>

<ParamField body="payment_method" type="string">Método sugerido (max 32).</ParamField>

<ParamField body="metadata" type="object">Dados livres (UTM, referrer, etc.).</ParamField>

<RequestExample>
```bash cURL
curl https://seu-dominio/api/v1/leads \
  -H "Authorization: Bearer aut_..." \
  -H "Idempotency-Key: lead-form-2026-05-05-abc" \
  -H "Content-Type: application/json" \
  -d '{
    "checkout_id": 5,
    "name": "João Silva",
    "email": "joao@example.com",
    "phone": "+5511999999999",
    "metadata": {
      "utm_source": "facebook",
      "utm_campaign": "blackfriday"
    }
  }'
```
</RequestExample>

<ResponseExample>
```json 201
{
  "data": {
    "id": 100,
    "checkout_id": 5,
    "name": "João Silva",
    "email": "joao@example.com",
    "phone": "+5511999999999",
    "status": "pending",
    "created_at": "2026-05-05T18:00:00+00:00"
  }
}
```
</ResponseExample>

<Note>
  IP e User-Agent são capturados automaticamente da request — não envie no body.
</Note>
